# Network Intrusion Detection System
FYP-26-S1-20  

## Project Scope  
The purpose of this project is to develop a virtualized dashboard system that allows a network administrator to easily monitor network activity using intrusion detection systems (IDS) such as Snort, Kismet, Zeek (Bro), and Suricata.  

Typically, data captured and filtered by these IDS tools will be displayed on a centralized dashboard for analysis. The system aims to provide visual representations of the analyzed data using data mining techniques or other relevant methods to support better decision-making.



## Overview  
This project aims to build a lightweight and modern intrusion detection dashboard to simplify the analysis of logs generated by IDS tools. The system supports multiple IDS platforms, including Snort, Suricata, Zeek, and Kismet, and presents their logs in a unified and user-friendly interface.  

Research was conducted on existing IDS monitoring tools such as Grafana, Security Onion, Snorby, and Graylog. While these tools are powerful, they often require complex configurations, dedicated hardware, or advanced technical knowledge. Some are designed mainly for on-premises environments, while others lack modern usability features.  

Based on these findings, this project proposes a dashboard that works with minimal configuration, supports remote monitoring through a hybrid cloud approach, and provides insights without requiring complex queries.  

The project operates under several constraints, including limited system resources, cross-platform compatibility, and support for multiple IDS log formats. It assumes that IDS logs are available and that network connectivity exists for optional cloud features.  

The scope of the project includes:
- Log collection  
- Data visualization  
- Alert handling  
- Notification system  

This project does not cover packet capture or the development of IDS rules.

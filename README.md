# Network-Intrusion-Detection-System
FYP-26-S1-20

# Project Scope 
The purpose of the project is to develop a virtualized dashboard system that allows a network administrator to easily monitor the network through the intrusion detection system (IDS) such as Snort, Kismet, Zeek(Bro), Suricata. Typically, all related data that filter by the IDS will be shown on the screen and waiting for the administrator to analyse them. This system should provide some visual presentations that result of data analysis based on the data mining techniques or any other related techniques on the captured data.

# Overview
This project aims to build a modern lightweight intrusion detection dashboard that makes it easier to analyze logs generated by intrusion detection systems (IDS). The system will support multiple IDS tools, including Snort, Suricata, Zeek, and Kismet, and display their logs in a single, easy-to-use interface.
Research was carried out on existing IDS monitoring and visualization tools such as Grafana, Security Onion, Snorby, and Graylog. Although these tools are powerful, many of them require complex setup, dedicated hardware, or advanced technical knowledge. Some are designed mainly for on-premises security operations centers, while others are outdated or lack modern usability and security features.
Based on this research, the project proposes a dashboard that works out of the box with minimal configuration. It supports remote monitoring using a hybrid cloud approach and provides useful insights without requiring users to write complex queries.
The main constraints of the project include limited system resources, the need to run on different operating systems, and support for multiple IDS log formats. The project assumes that logs from the supported IDS tools are available and that basic network connectivity exists for optional cloud features. It also depends on external threat intelligence services and predefined IDS log formats.
The project scope focuses on log collection, visualization, alert handling, and notifications. It does not cover packet capture or the creation of IDS rules.
